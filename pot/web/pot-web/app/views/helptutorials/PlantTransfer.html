Plant Transfer Request

All the Plant Transfer Requests from Resources Module are fetched at one place in this page for ease of use.

You can perform the following in the Plant Transfer Request Section:

  • Search the Existing Records
  • View the Plant Transfer Request details
  • Create a New Plant Transfer Request

Search Existing Records:

This section helps you to search the existing records.

To search the existing records, do the following.

  1. Select From Date and To Date.
  2. Select Login User Records to search the records of the logged-in user (or select All User Records to search the records of all the users).
  3. Click Search available on the right side of the screen.
  4. The search results are shown on the screen as shown in the figure.

Note:  

  • Click Reset to reset the screen to the default view.

You have successfully searched the existing records.


View Plant Transfer Request details:

This section helps you to view the Plant Transfer Request details.

To view the Plant Transfer Request details, do the following.

  1. Search the existing records. (See the Search Existing Records Section above)
  2. Click and the Requisition window opens as shown in the figure.

You have successfully viewed the Plant Transfer Request details.

Create New Plant Transfer Request:

This section helps you to Create a New Plant Transfer Request.

To Create a New Plant Transfer Request, do the following.

  1. Click and the Requisition window opens as shown in the figure.

  1. Select Origin Project and Destination Project.

Note:

  • Date Of Request is auto filled with the system date. 
  • Requisition ID, and Notification ID are auto generated after submitting Request.
  • Approval Date, and Approver ID are auto-filled after completion of the Approval process.
  • The plant(s) are transferred from Origin Project to Destination Project.
  • Logged in User details are auto-filled on the Requestor Sub-window.
  1. On the Approver Sub-window, select Employee ID, and the other text fields are auto-filled (Select from users list assigned to the selected destination project. The selected employee is the Approver for the Request).
